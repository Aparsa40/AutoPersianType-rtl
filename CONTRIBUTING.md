# Contributing to AutoPersianType / مشارکت در AutoPersianType

خوش‌آمدید! از علاقه‌مندی شما برای کمک به AutoPersianType متشکریم.
Welcome! Thank you for wanting to contribute to AutoPersianType.

---

## Workflow / روند کار پیشنهادی

1. **ایجاد Issue / Create an Issue**
   - قبل از ارسال Pull Request، لطفاً یک Issue باز کنید تا بحث انجام شود.
   - Before sending a Pull Request, please open an Issue to discuss the change or bug.

2. **Fork و Branch / Fork & Branch**
   - پروژه را Fork کنید و یک Branch جدید از `main` بسازید:
     مثال: `feat/my-feature` یا `fix/issue-123`
   - Fork the repository and create a branch from `main`:
     Example: `feat/my-feature` or `fix/issue-123`

3. **چک و تست / Run checks and tests**
   - قبل از ارسال PR، مطمئن شوید که همه تست‌ها و lint اجرا شده‌اند:

   ```bash
   npm run check
   npm run test
   ```

   - Make sure all linters and tests pass before opening a PR.

   - Make sure all linters and tests pass before opening a PR.

4. **ارسال Pull Request / Open a Pull Request**
   - توضیحات واضح و مراحل تست تغییرات را اضافه کنید.
   - Add a clear description and testing steps for your changes.

---

## نکات مهم / Important Notes

- فایل‌های هسته‌ای پروژه (`server/`, `package.json`, `drizzle.config.ts`, `vite.config.ts`) **تحت حفاظت هستند** و تغییر آن‌ها نیازمند تایید نگهدارندگان است (دیدن `.github/CODEOWNERS`).
- Core files (`server/`, `package.json`, `drizzle.config.ts`, `vite.config.ts`) are protected and require maintainers’ approval.

- برای مسائل کوچک، مستقیماً Fork کنید، تغییر دهید و Pull Request ارسال کنید.
- For minor issues, you can fork, change, and submit a PR directly.

- برای مسائل حساس یا امنیتی، لطفاً به `SECURITY.md` مراجعه کنید.
- For sensitive/security issues, see `SECURITY.md`.

---

## PR Checklist / چک‌لیست PR

- اجرای lint و چک‌ها: `npm run check`
- اجرای تست‌ها: `npm run test`
- اضافه کردن مستندات در صورت نیاز / Add documentation changes if needed
- اضافه کردن اسکرین‌شات برای تغییرات UI / Add screenshots for UI changes
